# Ship PR Workflow

Complete git workflow: branch → commit → PR → squash merge → cleanup.

## Arguments

- `$1` — Feature branch name (e.g. `feat/my-feature`). If omitted, derive from staged diff summary.
- `$2` — PR title. If omitted, derive from commit message.

## Steps

### 1. Validate Working State

```
!`git status --short`
!`git diff --stat`
!`git diff --cached --stat`
```

Confirm there are staged or unstaged changes. If the working tree is completely clean with no changes, abort and inform the user.

### 2. Identify Default Branch

```
!`gh repo view --json defaultBranchRef --jq '.defaultBranchRef.name'`
```

Store this as the target base branch for the PR and final checkout.

### 3. Create Feature Branch

- If `$1` is provided, use it as the branch name.
- Otherwise, inspect the diff and generate a concise kebab-case branch name prefixed with `feat/`.
- Run: `git checkout -b <branch-name>`

### 4. Stage All Changes

```
git add -A
```

Stage everything (staged + unstaged).

### 5. Commit

- Derive a conventional commit message from the diff if `$2` is not provided.
- Format: `<type>(<scope>): <short description>`
- Run: `git commit -m "<message>"`

### 6. Push Branch

```
git push -u origin <branch-name>
```

### 7. Raise PR Against Default Branch

```
gh pr create \
  --base <default-branch> \
  --head <branch-name> \
  --title "<pr-title>" \
  --body "<auto-generated summary of changes>"
```

- PR body must summarise what changed and why (derived from diff).
- Do NOT use `--draft` unless the user explicitly asks.

### 8. Squash Merge the PR

```
gh pr merge <pr-number> \
  --squash \
  --delete-branch \
  --subject "<squash commit message>"
```

- Use the PR title as the squash commit subject.
- `--delete-branch` removes the remote feature branch automatically.

### 9. Cleanup Local Branch & Pull Latest

```
git checkout <default-branch>
git pull origin <default-branch>
git branch -d <branch-name>
```

Delete the local feature branch and fast-forward the default branch.

### 10. Confirm

Print a short success summary:

- PR number and URL
- Squash commit SHA on default branch
- Current branch (should be default)
