# Global Claude Code Instructions

This file provides baseline instructions for all projects. Project-level CLAUDE.md files take precedence over these global settings.

## Writing style

- **Never use em dashes** (`—`, U+2014) in any output: code, comments, commit messages, PR descriptions, READMEs, emails, chat replies, generated docs — anywhere. This applies across every project.
  - Use commas, parentheses, colons, or two sentences instead. A hyphen-minus (`-`) is acceptable for ranges and compound modifiers; en dashes (`–`) should also be avoided.
  - When editing existing files, leave pre-existing em dashes alone unless the task is to update them; this rule governs *new* writing.
